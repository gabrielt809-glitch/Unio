# Privacy And Analytics

## 1. Estrategia futura de analytics

Analytics deve ser minimo, opt-in quando necessario e voltado a melhoria de produto. Primeiro estabilizar app, Supabase, testes e privacidade.

## 2. Cuidados de privacidade

O Unio lida com rotina, saude, habitos, sono, nutricao e financas. Esses dados sao sensiveis e nao devem ser enviados a ferramentas de analytics sem base clara, consentimento e minimizacao.

## 3. Dados que nao devem ser rastreados

- Titulos de tarefas.
- Notas.
- Valores financeiros.
- Categorias financeiras detalhadas.
- Dados de saude individuais.
- Email.
- User ID exposto fora do minimo operacional.

## 4. Eventos uteis no futuro

- App aberto.
- Modulo acessado.
- Fluxo concluido com sucesso.
- Erro por tipo tecnico, sem payload sensivel.
- Instalacao PWA detectada.

## 5. O que nao implementar ainda

- Session replay.
- Heatmaps.
- Envio de dados pessoais para analytics.
- Tracking sem documentacao e consentimento.
